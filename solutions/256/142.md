#### 双指针
**思路 S**
如果是环形，fast 和 slow 会相遇，此时：
+ 设 head 到入环点的长度 a，入环点到相遇点的长度 b，相遇点到入环点的长度 c；
+ 可得 `a + n * (b + c) + b = 2 * (a + b)`，推断出 `(n - 1)(b + c) + c = a`；
+ 即相遇时，从 head 开始出发和 slow 继续往下走，会在入环点相遇。

**复杂度分析**
+ 时间复杂度：O(n)，n 为链表长度，只遍历一次链表。
+ 空间复杂度：O(1)，只有两个指针，不需要存储其他变量。
